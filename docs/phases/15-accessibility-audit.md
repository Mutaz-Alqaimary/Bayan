# Phase 15 — Accessibility Audit

## Goal
Audit the entire platform built so far. Use the `a11y-auditor` subagent extensively here.

## Build / Audit
- WCAG awareness
- Keyboard navigation
- Focus management
- Screen reader support
- Accessible forms
- Accessible tables
- Accessible dialogs
- Color contrast validation
- RTL accessibility validation

## Docs to create
- `Accessibility.md` — what was audited, what was found, what was fixed, what remains

## Definition of done
- Every page/flow has been walked through with `a11y-auditor` and findings addressed or
  explicitly logged as known follow-up in `Accessibility.md`.
- RTL reading order and focus order verified, not just visual mirroring.
